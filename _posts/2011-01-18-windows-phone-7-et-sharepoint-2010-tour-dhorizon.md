<p><em><strong>Edit : </strong>Si vous vous conenctez depuis l’extérieur, il vous faut un serveur Forefront UAG</em></p>  <p>On parle beaucoup de Windows Phone 7 en tant que Téléphone, ou pour les développeurs, mais en parle peu de Windows Phone 7 pour les <em>Information Workers. </em>Cependant, Windows Phone 7 peut tout à fait s’intégrer dans une utilisation professionnelle : Connexion à Exchange, à l’annuaire Active Directory, et à Sharepoint 2010. Nous allons nous pencher sur ce dernier point.</p>  <p>&#160;</p>  <h2>Configurer Sharepoint 2010 pour Windows Phone 7</h2>  <p>Par défaut, lorsqu’on crée une web application pour héberger des collections de sites, on utilise une authentification Windows. Ce schéma d’authentification n’est pas supporté par le client Sharepoint pour Windows Phone 7. <strong>Il faut utiliser une “Claims Based authentication” avec Windows Basic authentication.</strong> Pour se faire depuis l’administration centrale : </p>  <ol>   <li>Créez une nouvelle Web application, </li>    <li>Choisissez <em>Claims Based Authentication</em>, </li>    <li>Dans les types d’authentification, choisissez <em>Basic autnehtication</em>. </li> </ol>  <p><u><strong>Attention</strong></u>: Passer d’une authentification Windows à une Claims based peut avoir des impacts sur une ferme existante. si vous voulez faire cela sur une ferme en production, je vous conseille de <a href="http://www.bewise.fr">vous tourner vers des spécialistes Sharepoint</a> :). </p>  <p>&#160;</p>  <p><em><a href="http://blog.maneu.org/wp-content/uploads/2011/01/WP7SP-0014.png"><img style="background-image: none;
